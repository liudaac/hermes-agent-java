🎯 核心特性

  • 多租户架构：每个租户拥有独立的配置、资源和安全策略
  • 多层沙箱隔离：文件、进程、网络、存储、线程池五层隔离
  • 细粒度权限控制：命令白名单、URL白名单、资源配额限制
  • 完整审计追踪：所有操作记录审计日志，支持合规要求
  • 多平台支持：Discord、Telegram、飞书、QQ等消息平台

🏗️ 架构层次

  • 接入层：GatewayServer处理HTTP/WebSocket请求，多平台Adapter统一接入
  • 核心层：HermesAgentV2引擎，ModelClient管理LLM连接，ToolRegistry工具管理
  • 隔离层：TenantContext封装各沙箱组件，提供统一的资源访问接口
  • 资源层：文件系统、子进程、网络连接、数据库存储

🔒 五层隔离机制

  • 文件沙箱：每个租户独立目录，路径限制防止目录遍历攻击
  • 进程沙箱：命令白名单+超时控制，防止危险命令执行和资源耗尽
  • 网络沙箱：URL白名单+速率限制，防止SSRF攻击和滥用
  • 存储配额：强制执行存储上限，流式写入追踪
  • 线程池隔离：租户级线程池，防止线程饥饿和OOM

工具执行流程（带资源限制）

sequenceDiagram
    autonumber
    participant Agent as HermesAgentV2
    participant Tool as Tool实现类
    participant TenantCtx as TenantContext
    participant Auth as 权限检查
    participant Quota as 配额检查
    participant Sandbox as 沙箱执行

    Agent->>Tool: execute(args, context)
    Tool->>TenantCtx: 获取租户上下文
    Tool->>Auth: 检查权限
    
    alt 权限检查失败
        Auth-->>Tool: SecurityException
    else 权限检查通过
        Tool->>Quota: 检查配额
        
        alt 配额超限
            Quota-->>Tool: QuotaExceededException
        else 配额充足
            Tool->>Sandbox: 在沙箱中执行
            Sandbox-->>Tool: 执行结果
            Tool-->>Agent: ToolResult.success
        end
    end
